MAVEN

maven目录结构

src
    -main
        -java
            -package
    -test
        -java
            -package
    resources

maven常用命令

mvn -v 查看maven版本
    compile 编译
    test 测试
    package 打包
    clean 删除target
    install 安装jar包到本地仓库中

创建目录的两种方式：
1.archetype:generate 按照提示进行选择
2.archetype:generate -DgroupId=组织名，公司网址的反写+项目名
    -DartifactId=项目名-模块名
    -Dversion=版本号
    -Dpackage=代码所存在的包名

pom解析

<modelVersion>指定当前pom的版本
<groupId>反写的公司网址+项目名
<artifactId>项目名+模块名
<version>版本号，第一个0表示大版本号，第二个0表示分支版本号，第三个0表示小版本号。
snapshot快照 alpha内部测试 beta公测 Release稳定版本 GA正式发布
<packaging>打包方式，默认是jar，还可以是war zip pom
<name>项目描述名
<url>项目地址
<description>项目描述
<developers>开发人员列表
<licenses>许可证信息
<organization>组织信息

<dependencies>
    <dependency>
        <groupId>
        <artifactId>
        <version>
        <type>
        <scope>依赖范围，只在什么范围内有用
        <optional>true或false，用来设置依赖是否可选，默认为false
        <exclusions>
            <exclusion>排除依赖传递列表

<dependencyManagerment>依赖管理
    <dependencies>
        <dependency>

<build>
    <plugins>插件列表
        <plugin>
            <groupId>
            <artifactId>
            <version>

<parent>子模块对父模块的继承
<modules>指定一起编译
